## Notas Generales

- Todos los tests requieren autenticación válida salvo casos específicos de registro/login.
- Las respuestas de error deben incluir código HTTP apropiado (400, 401, 403, 404, 409, 500, etc).
- Todos los timestamps deben estar en formato ISO 8601.
- IDs deben ser únicos y generados del lado del servidor.
- El versionado debe mantener historial completo sin corrupción.
- Las operaciones transaccionales deben ser atómicas.
- Los cambios deben registrarse con auditoría (quién, cuándo, qué cambió).
